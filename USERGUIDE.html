1 Overview

This tool provides two independent functions from a single page:

  • EDI Order File Generator — converts a subscription order spreadsheet into the fixed-width .txt format required by the TJClays POD Order Importer.
  • XML Metadata Generator — converts a simple spreadsheet of journal titles into individual XML specification files, ready for production workflows.

The two tools are entirely independent. You can use either or both in the same session without one affecting the other.

The left sidebar is divided into three panels: Import Excel Data (EDI orders), File Settings (collapsible batch configuration), and Generate XML Metadata. The main content area on the right contains a tab bar, stats bar, and the output view.

2 Before You Start

You will need the two Excel templates. Download them from within the app using the buttons at the bottom of each panel:

  • Download Order Template → saves order_file.xlsx
  • Download Metadata Template → saves metadata_template.xlsx
⚠️
Always use these files as your starting point. Do not rename, reorder or add columns — the app validates the file structure on upload and will reject any file that doesn't match the template exactly.

Row grouping: Rows that share the same Order Ref, Delivery Company and Delivery Name are automatically combined into a single order with multiple line items. You don't need to do anything special — just enter the rows and the app handles the grouping.

Step 2 — Upload the File

Drag your completed spreadsheet onto the Import Excel Data upload zone, or click the zone to browse for the file.

If the file passes validation, a green confirmation message appears showing the row count, and the Source Data tab populates with a preview table of your data.

If the file is rejected, a red error panel lists exactly which columns don't match the template — see the Troubleshooting section for details.

Step 3 — Check File Settings (optional)

Click the File Settings panel header to expand it. You can review or update the batch configuration before generating.

Steps 4 & 5 — Generate and Download

Click Generate EDI File. The app groups rows into orders, builds all EDI records, and switches to the EDI Preview tab.

The stats bar at the top shows a summary of what was generated. The preview is colour-coded by record type:

  • Purple — $$HDR file header and $$EOF footer
  • Blue — H1 order header records
  • Teal — H2 customer address records
  • Amber — H3 payment terms records
  • Green — D1 line item records

Click Download .txt to save the file. The filename follows the format:

PO.{fileId}_{time}_({date}).txt
e.g. PO.0027816_1423_(24-03-26).txt

This file is ready to submit to the BLOUK distribution system.

💡
Click Clear Data to reset everything and start a fresh upload.

4 Part 2 — Generating XML Metadata Files

Step 1 — Fill in the Metadata Template

Open metadata_template.xlsx and enter your journal data below the header row. One row per title. The template has just three columns:

ColumnWhat to enter
ISSNThe 13-digit ISSN — numbers only, no spaces or hyphens
TitleThe full journal or issue title
Page ExtentTotal number of pages. Drives the paper type selection and spine size calculation.

Everything else — trim size, paper type, spine width, GSM, colour, binding and lamination — is calculated automatically.

Step 2 — Upload and Validate

Drag your completed spreadsheet onto the Generate XML Metadata upload zone, or click to browse. The app checks two things:

  • Column structure — the file must match the template exactly
  • ISSN format — every ISSN must be exactly 13 digits with no spaces or hyphens

If either check fails, the file is rejected and a detailed error message shows which rows or columns need correcting. Fix the issues and re-upload.

Step 3 — Generate and Download

Click Generate XML & Download ZIP. Two files download automatically:

  • metadata.zip — contains one .xml file per row, named by ISSN (e.g. 9771472645051.xml)
  • metadata_summary.txt — a plain-text report listing every file generated with all the calculated values

What the App Calculates for You

Based on the Page Extent you enter, the app automatically determines all remaining specification values:

ValueRule
Paper TypeMagno Matt 130 gsm if Page Extent is 38 pages or fewer
Magno Matt 90 gsm if 39 pages or more
GSM130 or 90 — derived from paper type, written as a separate XML element
Spine SizeCalculated from page extent and paper GSM, rounded to the nearest whole millimetre
Trim HeightAlways 245 mm
Trim WidthAlways 170 mm
ColourAlways Mono
Binding StyleAlways Limp
LaminationAlways Matt

Each XML file follows this flat structure:

<?xml version="1.0" encoding="UTF-8"?>
<record>
  <isbn>9771472645051</isbn>
  <title>[2026] 1 FCR 4</title>
  <trim_height>245</trim_height>
  <trim_width>170</trim_width>
  <extent>120</extent>
  <spine>6</spine>
  <paper>Magno Matt 90 gsm</paper>
  <gsm>90</gsm>
  <colour>Mono</colour>
  <binding_style>Limp</binding_style>
  <lamination>Matt</lamination>
</record>

Click Clear to reset the metadata panel and upload a new file.

5 Troubleshooting

❌  The file was rejected — "Column mismatch"

Your spreadsheet doesn't match the expected template. The error panel shows exactly which columns are wrong, including the column number, the expected header, and what was found. The most common causes are:

  • Renamed column headers
  • Extra or missing columns
  • Columns in the wrong order

Download a fresh copy of the template using the Download button in the relevant panel and re-enter your data from scratch.

❌  The file was rejected — "Invalid ISSN"

One or more ISSNs in your metadata spreadsheet don't meet the format requirements. The error message identifies the affected row numbers. ISSNs must be exactly 13 digits — remove any spaces, hyphens or other characters before re-uploading.

The Download button doesn't appear

You need to click Generate EDI File (or Generate XML & Download ZIP) before the download button becomes available. Uploading the file alone is not enough — generation must complete first.

The template download button doesn't work

The template files (order_file.xlsx and metadata_template.xlsx) must be present in the same folder as the application files. Contact your system administrator if they are missing.

The page looks wrong or features aren't working

This app requires a modern browser. Use the latest version of Chrome, Edge or Firefox. Internet Explorer is not supported. If using Chrome, ensure you're on version 90 or later.

Copy to clipboard doesn't work

The clipboard API requires the page to be served over HTTPS or from localhost. On a plain HTTP connection, copy-to-clipboard will silently fail — use the Download .txt button instead.
